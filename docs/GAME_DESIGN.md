# 遊戲企劃書：企業接班人養成術

## Context（背景與目的）

使用者要開發一款單機離線網頁遊戲，部署於 GitHub Pages，PC 遊玩、可存檔。玩家扮演小型企業總經理（接班人），透過處理內外部事件與決策，讓公司從小型成長為中型、最終成為上市大型企業。本文件為第一階段交付物「遊戲企劃」，經使用者同意後才進入第二階段（程式架構、開發排程、資源分配），再進入開發與測試。

已與使用者確認的三個核心方向：

- **時間推進**：事件回合制。無真實時鐘，處理完一個月的事件即推進一個遊戲月；「1 小時 = 1 個月」僅為建議節奏，不強制。
- **遊戲期限**：累計 60 個遊戲月（= 遊戲中 5 年）即結束，不看真實日曆，沒玩的日子進度凍結、無損失。
- **難度**：預設普通難度、給一般成人玩的經營模擬；決策樹與難度參數全部外部化為 JSON，可調成簡單／普通／困難／自訂。

---

## 1. 核心循環（Core Loop）

```
月初簡報（上月結算結果、本月預告）
   ↓
本月事件逐一跳出（件數依難度與規模 3~16 件，來自輪替的內外部單位）
   ↓  玩家可隨時切換報表中心／新聞面板查資料再決策
全部事件決策完畢 → 按「結束本月」
   ↓
月底結算（套用公式、生效延遲效果、更新所有數據）
   ↓
自動存檔 → 檢查升級／降級／倒閉／上市 → 下一個月
```

一局 = 60 個遊戲月。提前達成「上市」為勝利結局之一；倒閉為失敗提前結束；60 個月到期依統計評分。

## 2. 企業規模衡量標準（內部 + 外部數據）

### 2.1 規模三級制

| 規模 | 名稱 | 象徵 |
|------|------|------|
| 第 1 級 | 小型企業 | 一層辦公室、10~49 人 |
| 第 2 級 | 中型企業 | 整棟小樓、50~199 人 |
| 第 3 級 | 大型企業（上市） | 企業總部大樓、200 人以上 |

### 2.2 核心數據（KPI）

**內部數據（7 項）**

| 數據 | 說明 | 範圍 |
|------|------|------|
| 現金 | 生存線，見底且借不到錢即倒閉 | 元 |
| 淨值 | 資產減負債，升級主門檻 | 元 |
| 月營收 | 升級主門檻 | 元 |
| 月淨利 | 連續獲利／虧損月數是升降級條件 | 元 |
| 員工數 | 升級硬門檻，人事成本來源 | 人 |
| 產品競爭力 | 研發累積成果，驅動市占 | 0~100 |
| 員工士氣 | 影響效率、良率、離職率 | 0~100 |

**外部數據（6 項）**

| 數據 | 說明 | 範圍 |
|------|------|------|
| 市占率 | 營收公式核心變數 | 0~100% |
| 品牌知名度 | 影響市占成長上限與單價空間 | 0~100 |
| 客戶滿意度 | 影響回購（需求穩定度）與客訴事件機率 | 0~100 |
| 銀行信用 | 決定融資額度與利率 | 0~100 |
| 股東信心 | 過低觸發撤換總經理危機；上市必要條件 | 0~100 |
| 政府關係／合規 | 重大違規擋上市；影響補助與稽查事件 | 0~100 |

輔助數據（不直接顯示於頂欄，見報表）：產能、產能利用率、良率、平均單價、原料成本率、離職率、供應商關係、通路關係、負債與月付利息。

### 2.3 升級／降級標準（預設值，全部可在 config 調整）

**小型 → 中型（需同時滿足，維持 2 個月）**
- 月營收 ≥ 800 萬
- 淨值 ≥ 3,000 萬
- 員工數 ≥ 50
- 連續 3 個月獲利

**中型 → 大型（上市）（需同時滿足，並通過「上市審查」事件鏈）**
- 月營收 ≥ 5,000 萬
- 淨值 ≥ 3 億
- 員工數 ≥ 200
- 連續 6 個月獲利
- 股東信心 ≥ 70、政府合規 ≥ 60、品牌 ≥ 60
- 無未解決的重大違規事件

**降級（任一成立，維持 2 個月）**
- 連續 4 個月虧損，或淨值跌破該級門檻的 60%，或員工數跌破門檻

**倒閉（立即結束）**
- 現金 < 0 且銀行拒絕紓困（信用過低或已達額度上限）

**規模改變時，公式常數隨之改變**（詳見第 5 節）：市場需求基數變大、管理費率上升、銀行額度放大、事件池換組、辦公室場景圖升級。

## 4. 事件系統與決策樹

### 4.1 資料驅動架構（動態難度的核心）

所有事件與參數外部化為 JSON，改資料即改難度與內容，不動程式碼：

- `data/events.json` — 事件庫（決策樹本體）
- `data/balance.json` — 經濟公式常數、各規模級常數組、升降級門檻
- `data/difficulty.json` — 難度檔（簡單／普通／困難）的倍率覆蓋
- `data/npcs.json`、`data/news.json` — NPC 資料與新聞模板

**單一事件的 schema：**

```
{
  id, dept(來源單位), tier(適用規模 1/2/3),
  trigger: { weight(抽取權重), conditions(數據區間/前置事件/月份範圍), once(是否僅一次) },
  title, text(情境敘述), speaker(NPC),
  options: [
    {
      label, 
      effects:  [ { var, delta 或 formula, delayMonths(0=立即) } ],
      random:   [ { chance, effects, resultText } ]   // 機率性結果
      followUp: eventId or null                        // 事件鏈
    }, ...(2~4 個選項)
  ]
}
```

### 4.2 每月事件的產生方式（輪替 + 權重）

- 每月抽取件數由「難度 × 企業規模」矩陣決定（定義於 difficulty.json，區間內隨機），規模愈大、難度愈高，需要處理的事件愈多：

| 每月事件數 | 小型（tier 1） | 中型（tier 2） | 大型（tier 3） |
|-----------|---------------|---------------|---------------|
| 簡單 | 3~5 | 5~7 | 7~9 |
| 普通 | 4~6 | 6~9 | 9~12 |
| 困難 | 5~8 | 9~12 | 14~16 |

- 設計基準：一個遊戲月的建議遊玩時間約 1 小時。困難 × 大型企業以每月約 15 件為中心值，等於平均每 4 分鐘做一次決策——模擬真實總經理「每週 3~4 項重要決策」（× 4 週）的壓力。

- 抽取時以「單位輪替保底」保證每個單位每 3 個月至少登場一次，其餘名額按權重與觸發條件抽取。
- 事件分四類：**例行決策**（低風險日常）、**機會**（高報酬需投入）、**危機**（不處理必惡化）、**連鎖劇情**（跨多月的事件鏈，如上市審查、勞資風暴）。
- 危機事件若選「擱置」，會在 1~2 個月後以更嚴重版本回歸。

### 4.3 決策樹展開（每單位 2 個完整範例；完整事件庫於開發階段產出）

以下展示決策樹深度與影響因素的寫法。目標事件庫規模：**每個內部部門 12~15 件、每個外部單位 10~12 件，另加連鎖劇情 6 條，共約 140~170 件**，依 tier 分層、tier 2/3 加重配置（高難度高規模時每月事件多）。搭配兩個防重複機制：例行決策類事件可重複觸發但**參數隨機化**（金額、幅度、對象在範圍內變化）；機會與危機類設定**冷卻月數**，短期內不重抽同一事件。

**研發部 R-01「下一代產品的技術路線」（tier 1，機會）**
- 沈技安：「現有產品再撐一年就會被淘汰，我們該押哪條路？」
- A. 自主研發（-300 萬現金分 3 個月扣；6 個月後產品競爭力 +15，30% 機率額外 +10 突破）
- B. 授權國外技術（-500 萬現金立即；3 個月後競爭力 +12，之後每月權利金使原料成本率 +2%）
- C. 維持現狀（無成本；競爭力每月 -1 持續 6 個月）

**研發部 R-07「競爭對手挖角研發主力」（tier 1~2，危機）**
- A. 加薪留人（人事成本 +5%/月；士氣 +5）
- B. 讓他走（競爭力 -8；40% 機率 2 個月後觸發「技術外流」後續事件）
- C. 反挖對方團隊（-400 萬；60% 成功競爭力 +10、失敗士氣 -5 且登上負面新聞）

**生產部 P-03「產線瓶頸」（tier 1，例行）**
- A. 買新設備（-600 萬現金、產能 +25%，2 個月後生效；折舊使月成本 +12 萬）
- B. 加班趕工（產能 +10% 立即；士氣每月 -3、良率 -2，持續到取消）
- C. 外包代工（產能 +20% 立即；原料成本率 +5%、20% 機率品質客訴事件）

**生產部 P-09「品質事故：整批瑕疵品流出」（tier 2，危機）**
- A. 全面回收並公開道歉（-800 萬；客戶滿意度 +5、品牌 -3 短痛，合規 +5）
- B. 私下換貨壓下來（-200 萬；35% 機率 2 個月後媒體爆料 → 品牌 -20、政府稽查事件鏈）
- C. 歸咎供應商（供應商關係 -20；50% 機率原料漲價報復）

**行銷業務部 M-02「年度廣告預算」（tier 1~3，例行，每年觸發一次）**
- A. 大打品牌廣告（-每月營收的 8%，持續 3 個月；品牌 +8、市占 +1.5%）
- B. 精準數位行銷（-4%；市占 +1%、客戶滿意度 +3）
- C. 縮減預算（省錢；品牌每月 -1、市占成長上限下修）

**行銷業務部 M-08「大客戶要求獨家供貨」（tier 2，機會＋風險）**
- A. 接受（市占 +4%、營收 +15%；通路關係 -15、單一客戶依賴 → 之後「白買家殺價」事件機率上升）
- B. 婉拒（客戶滿意度 -5；無其他損失）
- C. 給半年獨家（市占 +2%、營收 +8%；6 個月後自動觸發續約談判事件）

**人事部 H-01「基層集體反映薪資過低」（tier 1，危機）**
- A. 全面調薪 5%（人事成本 +5%；士氣 +10、離職率 -2%）
- B. 只調關鍵人員（成本 +2%；士氣 +3，15% 機率 1 個月後爆發不滿 → 士氣 -8）
- C. 精神喊話（免費；士氣 -5、離職率 +2%）

**人事部 H-06「要不要導入教育訓練制度」（tier 1~2，機會）**
- A. 完整制度（-150 萬 + 每月 30 萬；3 個月後良率 +3、競爭力 +3、士氣 +5）
- B. 只送主管受訓（-40 萬；士氣 +1、良率 +1）
- C. 不辦（無效果；tier 2 之後「管理失能」事件權重 +）

**財務部 F-02「應收帳款積壓」（tier 1，例行）**
- A. 給折扣催收（現金 +400 萬；營收 -3% 當月）
- B. 應收帳款融資（現金 +500 萬；月利息 +8 萬、銀行信用 -3）
- C. 不處理（30% 機率 2 個月後「客戶倒帳」：現金 -600 萬）

**財務部 F-07「閒置資金運用」（tier 2，機會）**
- A. 定存（每月 +利息微量，零風險）
- B. 投資理財商品（60% 每月 +40 萬；20% 不賺不賠；20% 一次 -500 萬）
- C. 加碼研發／行銷（轉入該部門，效果 ×1.2）

**大股東 S-01「季度業績質詢」（每季自動，例行）**
- 依上季獲利自動評價：達標 → 股東信心 +5；不達標可選：
- A. 坦承並提改善計畫（信心 -3；下季達標則 +10）
- B. 美化說法（50% 信心 +2；50% 被識破 -10）
- C. 請求增資支持（現金 +2,000 萬；信心 -5、之後質詢標準提高）

**大股東 S-05「董大川想安插親戚當副總」（tier 2，危機）**
- A. 接受（信心 +8；士氣 -8、每月管理費 +20 萬）
- B. 婉拒（信心 -10）
- C. 給閒職（信心 +3、管理費 +10 萬；10% 機率日後「親戚闖禍」事件）

**銀行 B-02「主動提供擴張貸款」（tier 1~2，機會）**
- A. 借 3,000 萬（現金 +3,000 萬；月利息依信用 15~30 萬、負債上升）
- B. 借 1,000 萬（同比例縮小）
- C. 不借（銀行關係 -2）

**銀行 B-06「景氣緊縮，銀行抽銀根」（tier 2~3，危機，信用 < 50 時權重大增）**
- A. 提前清償一半（現金大扣；信用 +10）
- B. 協商展延（信用 -5、利率 +1%）
- C. 轉向股東求援（股東信心 -8、現金 +貸款額）

**供應商 U-01「原料漲價 15%」（tier 1，危機）**
- A. 接受（原料成本率 +15%）
- B. 簽 2 年長約鎖價（僅 +5%；但期間出現更便宜供應商也不能換）
- C. 換新供應商（成本率 -5%；前 2 個月良率 -4、20% 機率斷料事件）

**供應商 U-04「石原料週轉不靈求預付」（tier 2，機會）**
- A. 預付半年貨款（現金 -900 萬；供應商關係 +25、原料成本率 -8% 持續一年）
- B. 拒絕（關係 -10；15% 機率 1 個月後斷料危機）
- C. 趁機壓價（成本率 -12%；關係 -25、日後危機事件中供應商不再幫忙）

**通路／客戶 C-03「白買家下年度大單但要求降價 12%」（tier 1~2，機會＋風險）**
- A. 接受（營收 +20%、單價 -12% → 淨利率被壓縮）
- B. 談判折衷（60%：-6% 成交、營收 +12%；40% 破局：客戶滿意度 -10）
- C. 拒絕（市占 -2%；品牌 +2「有骨氣」）

**政府 G-02「環保法規加嚴」（tier 2，危機，全產業）**
- A. 立即投資防治設備（-1,200 萬；合規 +15、之後稽查免疫 2 年）
- B. 分期改善（-每月 100 萬 × 12；合規 +8）
- C. 觀望（25%/月 機率被稽查：罰款 -500 萬、合規 -15、品牌 -5）

**政府 G-05「產業升級補助開放申請」（tier 1~2，機會）**
- A. 認真準備申請（-30 萬顧問費；合規 ≥ 50 則 70% 拿到 800 萬補助）
- B. 隨便送件（免費；20% 拿到）
- C. 不申請（無事）

**媒體 N-01「麻雀姐邀請專訪」（tier 1~3，機會）**
- A. 接受並開放參觀（品牌 +6；若本月有未解決品質／勞資危機 → 被寫出來，品牌 -10）
- B. 只做書面回覆（品牌 +2）
- C. 拒絕（麻雀姐好感 -；日後負面報導機率 +5%）

**連鎖劇情（6 條，跨多月）**：上市審查之路（tier 3 主線，5 階段）、勞資大風暴、技術外流疑雲、家族接班內鬥、金融風暴（全域負面 buff 3 個月）、殺手級新產品（研發競爭力 ≥ 80 觸發）。

### 4.4 新聞與小道消息系統

- 每月 2~4 條，顯示於新聞面板：**產業新聞**（預告 1~2 個月後的市場變動，可信）、**公司新聞**（玩家決策的後果見報）、**小道消息**（60% 真 40% 假，難度可調）——例如「聽說石原料要漲價」，信了先簽長約可避險，但也可能是假消息白花錢。
- 新聞是玩家「監控外部情勢」的主要情報來源，讀新聞有實質收益，構成監控玩法。

## 5. 經濟引擎（月結算公式）

```
市場需求   = 基準需求(依 tier) × 產業景氣係數 × (1 + 品牌/200)
銷售量     = min(市場需求 × 市占率, 產能 × 利用率 × 良率)
月營收     = 銷售量 × 平均單價
月成本     = 原料成本(銷售量×成本率) + 人事(員工數×平均月薪×士氣效率係數)
             + 行銷費 + 研發費 + 管理費(營收×管理費率,依 tier) + 利息 + 折舊
月淨利     = (月營收 − 月成本) × (1 − 稅率)
現金       += 月淨利 ± 融資/投資現金流
淨值       += 月淨利
市占率     += (產品競爭力 − 對手競爭力)/400 + 行銷效果 − 對手侵蝕(依 tier)
士氣/品牌/關係類 → 每月向 50 自然回歸 1 點（避免鎖死），事件效果另計
```

**tier 常數組（balance.json 內三組並列）**：基準需求（1:1 億／2:6 億／3:25 億月規模）、管理費率（8%／12%／15%）、對手侵蝕強度、銀行額度上限、平均月薪基準——升級後市場更大但管理更貴、對手更強，形成「規模愈大、賺得快也燒得快」的節奏。

**難度檔（difficulty.json）覆蓋項**：初始現金、每月事件數矩陣（難度 × tier，見 4.2 節）、危機事件權重倍率、負面效果倍率、升級門檻倍率、小道消息真實率、對手成長速度、選項是否顯示精確數字。

## 9. 使用者原案之外，本企劃補充的設計

1. 倒閉與被撤換兩種失敗結局（原案只有成長方向）
2. 貸款／增資等融資機制（現金見底前的緩衝與代價）

4. 小道消息真偽機制（讓「監控新聞」成為有價值的玩法而非裝飾）
5. 危機擱置會惡化回歸（防止玩家無腦跳過）
6. 數值自然回歸機制（防士氣／關係鎖死在極端值）
7. 上市為「事件鏈審查」而非只看數字（結局更有儀式感）
8. 隨機種子存檔、匯出匯入、決策即時存檔（單機遊戲的中斷保護）
9. 特殊成就與大事記回顧（提高重玩價值）
10. 選項不顯示精確數字（普通以上難度），玩家需從報表回饋中學習因果——這是「養成術」的核心樂趣
